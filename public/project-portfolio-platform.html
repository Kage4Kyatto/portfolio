Project Breakdown

Portfolio Website Platform

This project combines a static multi-page frontend with an Express backend, admin authentication, message handling, and deployment-oriented configuration.

Challenge

The Problem

Building a portfolio that demonstrates full-stack capabilities while remaining secure, maintainable, and showcasing best practices in modern web development.

  • Needed multi-layer security (authentication, rate limiting, XSS prevention)
  • Required message persistence and admin management
  • Had to work across multiple runtime environments
  • Demanded production-ready error tracking
Solution

The Approach

A polyglot architecture combining static content delivery with protected API endpoints, layered security middleware, and comprehensive monitoring.

  • Express server with Helmet security headers
  • Session-based admin authentication with OTP support
  • Per-endpoint rate limiting and input sanitization
  • Sentry error tracking for production monitoring
Results

Key Metrics

  • ✅ Zero critical security vulnerabilities
  • ✅ 100% test pass rate (9/9 tests)
  • ✅ Admin dashboard with analytics and filtering
  • ✅ Production-ready deployment configs
Stack

Core technologies

  • Frontend: HTML, CSS, JavaScript
  • Backend: Node.js and Express
  • Additional API path: PHP endpoints
  • Tooling: TypeScript utilities and tests
Highlights

Key implementations

  • Host-aware robots and sitemap generation
  • Protected admin page with credential checks
  • Contact form with fallback endpoint strategy
  • Message persistence in local JSON store
Quality

Stability approach

  • Node syntax lint checks
  • API tests for health and validation
  • Rate limiting for contact submission endpoint
  • Admin lockout for repeated failed auth attempts

Architecture

System Design Overview

The platform uses a layered approach with clear separation of concerns.

Security Layers

Defense in Depth

  • Transport: HTTPS with Helmet security headers
  • Authentication: Session-based with credential validation
  • Rate Limiting: Per-endpoint sliding window counters
  • Input Validation: XSS sanitization and email validation
  • Monitoring: Sentry error tracking and performance tracing
Feature Stack

Core Capabilities

  • Admin Dashboard: Real-time analytics and message management
  • Contact API: Form submission with validation and persistence
  • Blog System: Dynamic post management with pagination
  • Dark Mode: Client-side theme toggle with localStorage
  • Notifications: Toast system with accessibility support

Development Journey

Lessons Learned

Building this portfolio revealed key insights about web development practices.

What Worked Well

  • Layered validation (client + server) catches errors early
  • Test-driven development prevented regressions
  • Modular middleware design made security easy to reason about
  • Clear separation of concerns simplified debugging

Key Improvements

  • Image lazy loading reduced initial bundle size
  • Per-endpoint rate limiting is better than global limits
  • Schema.org structured data improves SEO and accessibility
  • Admin analytics reveal actual user patterns
